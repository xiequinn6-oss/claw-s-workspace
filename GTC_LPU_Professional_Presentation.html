Groq LPU的技术诱惑与架构挑战

核心问题

除了低时延推理，Groq的LPU的商业第一性是什么？AFD分离来了，PD分离过时了？Rubin这一代LPU用起来的架构挑战有哪些？

TLDR总结

本文从GTC Keynote推理性能图的细节出发，解读LPU的价值与取舍。Groq LPU是一颗用算力和容量换极致带宽的确定性执行芯片，用三星4nm生产，完全不依赖台积电和内存厂的产能。

白话版：

  1. 因为产能供不应求，在不依赖台积电，不和Rubin抢wafer和memory的前提下，NV额外用三星的产能做了Groq LPU。
  2. 因为没用memory，Groq LPU用SRAM堆了巨大的带宽，代价是巨小的算力和容量。
  3. 因为容量小算力小，Groq针对低时延场景把decode拆分成attention和ffn，这种部署叫AFD。
  4. AFD每层要2次通信，1次通信要7跳，跳来跳去时延大很难受。
  5. AFD利用率也是问题，需要一个全新的调度框架提升利用率。

关键发现

LPX的价值定位

左边市场（50-100 TPS）：Rubin NVL72独立即可满足

右边市场（200-400 TPS）：Rubin单独吞吐急剧下降，而Rubin + LPX组合维持了高性能

性能对比：在400 TPS下，Rubin + LPX比Blackwell高35倍的吞吐效率

隐藏的两个关键细节

细节一：Rubin + LPX曲线几乎是平的

从200 TPS到400 TPS，吞吐几乎没有下降。而Rubin NVL72随着TPS/User提高急剧下降。

技术原因：GPU在高TPS要求小batch，小batch算力利用率低；LPU的算力带宽比只有4，从一开始就没吃到大batch的算力收益

细节二：右半边是2T模型，LPU的容量撑不住

High/Premium档对应的是GPT MoE 2T模型。即使W4量化也要1 TB。

LPX整个机柜的SRAM只有128 GB（256颗LPU × 500MB），连1/8都放不下。

根本性权衡： LPU用算力和容量换来了极致带宽。35倍的TPS/MW提升背后，是算力天花板（平线）和容量缺口（装不下2T）。理解了这个trade-off，才能理解后面AFD架构为什么要这么拆。

芯片架构对比

GPU vs LPU：完全不同的设计哲学

GPU架构：通用计算优化

设计理念

GPU是为通用并行计算而设计，强调灵活性和适应性：

  • 多层次缓存体系：L1/L2/L3缓存 + HBM内存
  • 动态调度机制：运行时硬件调度
  • 乱序执行：填满流水线，提高利用率
  • 复杂控制逻辑：分支预测、重排序缓冲区等

GPU面积分配

  • ~40%：计算单元（CUDA Core, Tensor Core）
  • ~30%：缓存层次和控制逻辑
  • ~20%：互连和接口（NVLink, PCIe）
  • ~10%：其他（时钟、供电等）

LPU架构：推理专用优化

设计理念

LPU是为确定性AI推理而设计，强调低延迟和可预测性：

  • 单层SRAM存储：没有缓存层次，直接访问
  • 编译器静态调度：编译时决定一切
  • 确定性执行：消除运行时不确定性
  • 简化控制逻辑：去掉分支预测、乱序执行等

LPU面积分配

  • ~70%：SRAM存储（500MB）
  • ~15%：计算单元（MXM, VXM等）
  • ~10%：互连（C2C SerDes）
  • ~5%：控制逻辑和其他

核心差异： GPU用复杂性换取通用性，LPU用专用性换取确定性。GPU的大量面积用于"等"和"猜"（缓存、预测），LPU将这些面积全部换成SRAM，实现极致带宽。

集群及互联架构

四级带宽衰减与互连拓扑

4.1 ComputeTray（1U计算托盘）

  • 组成：8颗LP30 + 2颗Altera FPGA（FEL）+ Intel Granite Rapids CPU + BlueField-4网卡
  • 规格：SRAM合计4GB / 1.2 PB/s，FP8算力9.6 PFLOPS
  • Fabric Expansion Logic（FEL）：2颗FPGA挂载256GB DDR5，协议桥接 + 参数暂存

4.2 互连拓扑与四级带宽衰减

四级带宽衰减

带宽从内到外衰减14倍（5.6T → 400G）。GPU ↔ LPU的通信恰好走最外层的以太网——这个带宽层级直接决定了AFD的通信开销。

去向 Link 数 带宽 特点
Tray内全连接 56 5.6 Tbps C2C直连，确定性，延迟最低
Tray间（机柜内） 31 3.1 Tbps 背板走线，仍然是C2C
机柜间 4 400 Gbps 前面板出去 → 以太网，延迟和抖动显著增大
FEL FPGA 4 400 Gbps 参数加载 + 以太网桥接

通信延迟影响

  • Tray内通信：~100ns（理想）
  • 机柜间通信：~10μs（理想）到 ~100μs（实际）
  • 影响：AFD每层需要2次跨机柜通信，每次7跳，通信可能吃掉40%的耗时

架构挑战： LPU的高带宽优势在芯片内部，但跨机柜通信受限于以太网带宽。这是AFD分离架构必须面对的通信瓶颈，也是当前系统最大的性能限制之一。

确定性执行原理

从兜不确定性到消除不确定性

传统芯片的"不确定性兜底"

现代芯片架构的大部分复杂性，都是在兜住不确定性：

  • 多级缓存：赌数据在不在附近
  • 乱序执行：填住等待的空隙
  • 重排序缓冲区、指令队列、Store Buffer：在不确定的延迟下维持吞吐
  • 网络层面的拥塞控制、动态路由、仲裁器：应对数据包到达时间和路径的不确定性

本质： 用面积和复杂度换取对不确定性的容忍。它们很有效，但也意味着芯片的大量面积不是在"算"，而是在"等"和"猜"。

Groq的逆向思维：从源头消除不确定性

  • 单层Scratchpad SRAM：没有缓存层次，不存在cache miss
  • 编译器静态调度：编译时决定每个时钟周期的数据位置和指令
  • 确定性片间网络：编译时规划通信路径和时序，没有运行时争用
  • 硬件deskew指令：周期性校正多芯片时钟漂移，维持全局同步

面积重新分配

省下来的面积全部填了SRAM：

  • 不需要L1/L2/L3缓存
  • 不需要乱序执行单元
  • 不需要重排序缓冲区
  • 不需要指令队列
  • 不需要拥塞控制硬件

结果：LP30能在一颗die上塞500MB、做到150TB/s带宽

代价：灵活性 模型架构变了就要重新编译。大模型架构迭代速度极快，LPU的先编译架构能否跟上TensorRT-LLM的适配节奏，决定了400-500万颗LPU是"卖出去就能用"，还是"卖出去了但用不起来"。

确定性执行 vs 传统执行

AFD分离与三大挑战

从PD分离到AFD分离的演进

5.1 从PD分离到AFD分离

PD分离（Prefill-Decode分离）

原理：Prefill（上下文编码）和Decode（自回归生成）对延迟的敏感度不同

部署：Prefill用GPU（高吞吐），Decode用LPU（低延迟）

基础：SLA（服务等级协议）差异

AFD分离（Attention-FFN-Decode分离）

原理：将Decode进一步拆分为Attention和FFN（前馈网络）

部署：

  • Attention：GPU（容量大，算力高）
  • FFN：LPU（带宽高，延迟低）
  • Decode：协调层

基础：必须靠异构硬件

5.2 挑战一：容量——SRAM放不下万亿模型

  • GPT MoE 2T模型：W4量化后约1 TB
  • LPX机柜SRAM：128 GB（256颗 × 500MB）
  • 缺口：仅能容纳1/8的模型参数
  • FEL DDR5的局限性：256 GB容量，~400 GB/s带宽，比SRAM慢375倍

5.3 挑战二：通信——每层两次跨机柜以太网

AFD通信模式

GPU（Attention） → LPU（FFN） → GPU（Attention） → ...

  • 每层：2次跨机柜通信
  • 每次通信：7跳（GPU HBM → GPU内存控制器 → PCIe → CPU → 以太网卡 → 交换机 → LPU FEL → LPU SRAM）
  • 通信开销：在400 TPS（2.5 ms/token）场景下，通信可能吃掉40%的耗时

5.4 挑战三：调度——BSP失效带来的颠覆式需求

BSP（Bulk Synchronous Parallel）模型的失效

传统模式：所有芯片完成当前层 → 全局同步 → 进入下一层

AFD问题：GPU和LPU计算时间不同，强制同步造成资源闲置

全异步调度的需求

目标：让GPU和LPU各自以最快速度推进，通过流水线隐藏通信延迟

挑战：需要全新的调度框架和编译器支持

AFD分离架构与通信挑战

解决方案与展望

全异步调度 + 硬件升级

6.1 软件解法：全异步调度框架

核心思想

解耦：GPU和LPU不再强同步

流水线：GPU计算第N层Attention时，LPU计算第N-1层FFN

缓冲区：中间结果缓冲，平滑计算和通信的波动

技术挑战

  • 依赖关系管理：确保数据一致性
  • 资源调度：动态分配计算和通信资源
  • 容错机制：处理硬件故障和性能波动

6.2 硬件升级路径

短期（2026-2027）：FEL ASIC化

目标：用定制ASIC替换FPGA

收益：降低延迟，提高能效，增加带宽

中期（2027-2028）：C2C直连GPU-LPU

目标：绕过以太网，实现GPU和LPU的直接高速互连

技术：NVLink-C2C桥接芯片

收益：通信延迟降低10倍，带宽提高10倍

长期（2028+）：统一内存架构

目标：GPU HBM和LPU SRAM共享统一地址空间

技术：CXL 3.0+，内存语义网络

收益：彻底消除数据拷贝，实现真正的异构计算

6.3 结论：确定性计算的新范式

技术范式转移： Groq LPU代表了一种根本性的技术范式转移：从用复杂度兜住不确定性，到用确定性换掉复杂度。

商业逻辑重构

  1. 第一性：产能独立，不依赖台积电和内存厂
  2. 第二性：用算力和容量换极致带宽
  3. 第三性：确定性执行带来的服务质量保证

最终判断

LPU不是GPU的替代品，而是GPU的补充。在AI推理的Pareto前沿上，GPU+LPU的异构组合正在定义新的性能边界。成功的关键在于能否解决AFD分离带来的三大挑战，这需要芯片、软件、系统三个层面的协同创新。

Memory Architecture Revolution

SRAM as Primary Storage, Not Cache

Eliminating the Memory Wall

The Problem: Sequential Layer Execution

Inference presents sequential layer execution with much lower arithmetic intensity than training, exposing the latency penalty of DRAM/HBM access (100-200ns per fetch).

The Solution: On-chip SRAM

LPU integrates hundreds of megabytes of on-chip SRAM as primary weight storage (not cache), resulting in 1-2ns access latency (100├Ч faster than DRAM).

GPU DRAM Access: 100-200ns

LPU SRAM Access: 1-2ns

100├Ч latency reduction fundamentally changes the economics of AI inference.

This architectural choice represents a radical departure from decades of computer architecture convention that treated SRAM as cache and DRAM as primary storage.

Deterministic Execution Model

From Dynamic Scheduling to Compiler-Directed Execution

The Power of Predictability

Traditional GPU Execution

Dynamic hardware scheduling introduces non-deterministic latency due to cache misses, resource contention, and unpredictable memory access patterns.

LPU Execution Innovation

All scheduling decisions are made at compile time, not runtime. The compiler knows exactly when and where every operation will execute.

Scheduling Time Runtime (Hardware) Compile Time (Software)

Deterministic execution enables service level agreements (SLAs) for AI inference that were previously impossible with traditional architectures.

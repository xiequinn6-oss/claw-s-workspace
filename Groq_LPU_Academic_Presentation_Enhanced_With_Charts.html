1. The Memory Wall Problem

Shifting Bottleneck in AI Inference

Traditional von Neumann Architecture

Compute ← Memory bottleneck in LLM inference

  • Memory bandwidth limits token generation
  • HBM access latency: 100-200ns
  • Energy cost: ~200pJ/bit for DRAM access
ISCA 2020: "Deep learning workloads are increasingly memory-bound"

2.1 GPU Architecture: Complex Data Scheduling

Hierarchical Memory & Non-deterministic Execution (e.g., A100/H100)

Complex Memory Hierarchy

Data must travel through multiple unpredictable layers (HBM → L2 → L1) before reaching computation units, causing latency variations.

Non-Deterministic Scheduling

Hardware schedulers dynamically allocate threads. Cache hits/misses create unpredictable execution times (jitter) during LLM token generation.

Coherence Overhead

Maintaining data consistency across massive parallel cores requires complex coherence protocols, burning power and cycles.

Key Insight: GPU's complexity creates non-deterministic latency, unsuitable for real-time LLM inference.

2.2 LPU Architecture: Deterministic Data Flow

SRAM & Compiler-driven Linear Execution (Groq TSP)

No Caches, Only SRAM

Eliminates complex memory hierarchies. Massive on-chip SRAM provides immense bandwidth without cache-miss latency penalties.

Software-Defined Determinism

Hardware thread schedulers are removed. The compiler precisely orchestrates when every byte moves and computes, guaranteeing exact execution time.

Linear Data Flow

Data moves in a straight line through functional units. No arbitration, no contention, no unpredictable stalls.

Key Insight: LPU's simplicity enables deterministic latency, perfect for real-time LLM inference.

3. TSP Microarchitecture

Spatial VLIW Design

Architectural Innovation

4. Deterministic Execution Model

Eliminating Non-Determinism

Traditional GPU vs TSP

Component GPU TSP
Scheduling Hardware dynamic Compiler static
Memory Cache hierarchy SRAM only
Network Packet-switched Time-division

Determinism Benefits

  • Predictable Latency: No cache misses or contention
  • Linear Scaling: Multi-chip behaves as single core
  • Debugging: Reproducible execution traces
  • Quality of Service: Guaranteed performance SLAs
ISCA 2020: "Determinism enables precise performance prediction and scaling"

5. Compiler-Driven Data Routing

Software-Defined Architecture

Compiler Responsibilities

  1. Instruction Scheduling: Every cycle planned at compile time
  2. Data Movement: Exact routing through chip
  3. Resource Allocation: Functional unit assignment
  4. Network Timing: Multi-chip synchronization

Compiler Optimization Challenges

  • Complexity: NP-hard scheduling problem
  • Compile Time: Hours for large models
  • Model Specific: Recompile for each model
  • Toolchain Maturity: Early stage ecosystem

Trade-off: Runtime simplicity for compile-time complexity

6. Multi-Node Scaling (ISCA 2022)

Software-Defined Networks

Plesiosynchronous Protocol

  • Clock Synchronization: Cancel natural drift
  • Deterministic Network: Time-division multiplexing
  • Single Logical Core: 1000+ chips as one processor
  • Compiler Scheduling: Network traffic planned at compile time
ISCA 2022: "Software-defined tensor streaming multiprocessor enables linear scaling"

8. Commercial Implications

From Architecture to Value

Technical Advantages → Business Value

  • Determinism → SLA Guarantees: Predictable performance for enterprise
  • Low Latency → Real-time Apps: Chat, coding, gaming AI
  • Energy Efficiency → TCO Reduction: Lower operating costs
  • Linear Scaling → Future-proof: Add chips as needed

Market Positioning

  • Niche Focus: Inference-only specialization
  • Early Adopters: Real-time AI applications
  • Ecosystem Challenge: CUDA dominance
  • Long-term Play: Inference market growth

Strategic Insight: Groq is not competing with Nvidia on training, but creating new inference market

9. Conclusion & Future Work

Technical Contributions

Key Innovations (ISCA 2020/2022)

  1. Spatial VLIW Architecture: Eliminates control overhead
  2. Deterministic Execution: Predictable performance
  3. Compiler-Driven Design: Software-defined hardware
  4. Linear Multi-chip Scaling: Software-defined networks

Future Research Directions

  • Compiler Optimization: Reduce compile times
  • Model Support: Larger than SRAM capacity
  • Mixed Precision: FP8, INT4 quantization
  • New Workloads: Beyond transformers
